## shadcn/ui 초기화

```bash
pnpm dlx shadcn@latest init -y -f --css-variables -b radix -p nova
```

CLI 최신 버전에서 `--base-color` 플래그가 사라져서 여러 번 시도가 필요했어요. `radix` + `nova` 프리셋 조합을 찾는 데 좀 걸렸는데, 알고 나면 한 줄이에요. 역시 이지!

생성된 것들:
- `components.json` — shadcn 설정
- `src/lib/utils.ts` — `cn()` 유틸 함수
- `globals.css` — oklch 기반 CSS 변수 블록

## ESLint 마이그레이션 — 예상치 못한 삽질

`pnpm lint` 실행하자마자 에러가 뜨더라고요:

```
Invalid project directory provided, no such directory: .../lint
```

**원인:** Next.js 16에서 `next lint` 명령이 완전히 제거됐어요. 공식 codemod로 해결:

```bash
npx @next/codemod@canary next-lint-to-eslint-cli . --force
```

이게 `package.json`의 `"lint": "next lint"`를 `"lint": "eslint ."`로 자동 변경해줘요. 근데 생성된 `eslint.config.mjs`에 ESM 확장자 누락, `eslint-config-next` 버전 불일치(15 → 16) 같은 추가 이슈가 있었어요.

**교훈:** 메이저 버전 업그레이드 시 공식 마이그레이션 가이드를 먼저 확인하는 게 가장 빠릅니다. 삽질하다 알게 된 거지만요 ㅋㅋ

## 공통 레이아웃 3총사

과한 디자인은 나중으로 미루고, 최소 골격만 잡았어요:

| 컴포넌트 | 역할 |
|----------|------|
| `SiteHeader` | 로고 + 네비게이션 링크 5개 |
| `SiteFooter` | 저작권 + GitHub/소개 링크 |
| `PageContainer` | max-w-4xl 공통 래퍼 |

**설계 포인트:** `PageContainer`를 `RootLayout`에서 전체 감싸지 않고, 각 페이지가 직접 사용하는 방식으로 했어요. 나중에 풀너비 히어로 섹션 같은 예외를 유연하게 처리하려고요.
