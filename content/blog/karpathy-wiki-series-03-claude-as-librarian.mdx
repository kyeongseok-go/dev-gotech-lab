안녕하세요 고텍이예요! 한 주가 좀 지났네요.

그동안 vault에 페이지가 좀 쌓였어요. 4월 22일에 구조 잡고, 23일에 ingest 두 번 돌렸더니 벌써 20개 넘는 wiki 페이지가 생겼거든요. LY Corp MCP 아티클 하나에서 10개, GPT Image 2 영상 하나에서 7개. 이게 LLM Wiki의 맛이에요.

근데 1주일 굴리면서 제일 크게 배운 게 있어요. **Claude한테 "wiki 정리해줘"라고 하면 안 된다는 것.** 이유 설명할게요. 같이 고!!

## "wiki 정리해줘"가 왜 안 통하는가

처음 며칠은 그냥 "이거 정리해서 wiki 만들어줘"라고 했어요. 결과물이 그때그때 달랐어요.

어떤 날은 frontmatter 없이 만들고, 어떤 날은 sources 필드 빠트리고, 어떤 날은 raw/를 수정해버리고. Claude가 매 세션마다 "오늘 첫 대화"니까 context가 없으니까 당연한 거예요.

해결책은 **CLAUDE.md에 역할을 명문화**하는 거예요. 폴더별로.

## CLAUDE.md 계층 구조

Obsidian에서 Claude Code를 쓸 때, Claude는 현재 작업 디렉터리와 그 상위에 있는 CLAUDE.md를 자동으로 로드해요. 이 성질을 이용해서 폴더별 규칙을 격리할 수 있어요.

저는 이렇게 구성했어요:

```
/  (vault root)
├── CLAUDE.md          ← 최상위 운영 헌법
├── raw/
│   └── CLAUDE.md      ← "이 폴더는 읽기만 한다"
├── wiki/
│   └── CLAUDE.md      ← "이 폴더는 네가 쓰는 곳"
└── Output/
    └── CLAUDE.md      ← "결과물 초안 규칙"
```

루트 CLAUDE.md에는 Librarian 역할 정의, 3-layer 아키텍처, 10가지 핵심 규칙.
raw/CLAUDE.md에는 딱 한 가지: **절대 수정 금지**. 그리고 왜 금지인지.
wiki/CLAUDE.md에는 type 분류 기준, frontmatter 필수 필드, wikilink 규칙.

이렇게 해두면 Claude가 raw/ 안에서 작업할 때는 raw 규칙이 로드되고, wiki/에서 작업할 때는 wiki 규칙이 로드돼요. 컨텍스트 오염이 없어요.

## Librarian 메타포

루트 CLAUDE.md에 이렇게 적어뒀어요:

> "경석님은 소스 큐레이션·질문·방향 제시를 담당하고, Claude는 위키의 **관리자(Librarian)**로서 모든 잡일(요약·크로스 레퍼런싱·파일링·유지보수)을 담당한다."
>
> "경석님이 프로그래머라면 **Obsidian은 IDE, Claude는 프로그래머, wiki는 코드베이스**다."

이 메타포가 의외로 중요해요. "Claude는 저자·편집자"라고 박아두면 Claude가 저한테 "이건 어떻게 할까요?" 물어보는 대신 스스로 판단해서 파일링하거든요.

반대로 "절대 하지 말아야 하는 것"도 명문화하면:

- `raw/` 원본 수정 ← 금지
- 허락 없이 기존 문서 대규모 재편 ← 금지
- 출처 불명확한 정보 wiki에 삽입 ← 금지

이게 없으면 Claude가 선의로 틀린 행동을 해요. "정리해줄게요~" 하면서 원본을 수정해버리는 것처럼요.

## 맥락 상수 (Decision Heuristics)

CLAUDE.md에서 제일 공을 들인 섹션이에요.

Claude가 판단이 필요한 순간마다 참조하는 상수들이에요:

> - **커리어 조언** → **"1년 내 안정 대기업 취업"** 최우선 제약. 창업·프리랜싱 제안은 부차.
> - **콘텐츠 대상 독자** → **부모님 + AI 초심자** 1순위. 난도·톤 기준.
> - **브랜드 슬로건** → **"Go, Build the technology, more easier"** — 콘텐츠 톤의 북극성.
> - **회피**: 민폐 가능성, 취업 지연 요구, 엘리트주의 톤.

이게 없으면 Claude가 내 상황을 모르고 엉뚱한 조언을 해요. 예를 들어 "이 vault를 기반으로 RAG API 서버 만들어볼까요?" 같은 제안. 취업 준비에 집중해야 하는 사람한테 새 프로젝트를 제안하는 거잖아요.

맥락 상수가 박히면 Claude가 "1년 내 취업 최우선"을 알고, 제안의 우선순위를 알아서 조정해줘요.

## About_Me vs 나의_맥락_요약 분리

두 파일을 따로 만들었어요.

- **About_Me.md** — 외부용. 객관 이력, 프로젝트 목록, MBTI. 포트폴리오에 링크할 수 있는 것들.
- **나의_맥락_요약.md** — AI용. "1년 내 취업 최우선", 솔직한 약점, 전략 프레임. 외부에 보여주기 어려운 것들.

이 분리가 중요한 이유: About_Me는 "내가 어떤 사람인가"이고, 나의_맥락_요약은 "내가 어떤 판단 기준을 갖고 있는가"예요. Claude한테 필요한 건 후자예요.

새 Claude 세션을 열 때 "나의_맥락_요약.md 먼저 읽어줘"라고 하면 5초 만에 맥락이 공유돼요. 쉽죠?

## 1주 운영 회고

오늘 기준으로 wiki 페이지가 20개를 넘었어요. 처음엔 "Notion보다 복잡한 거 아냐?"라는 생각도 들었는데, 쌓이면서 달라지더라고요.

제일 크게 다른 점: **질문의 질**이 달라졌어요.

이전: "내 강점을 어떻게 어필해야 해?"
지금: "티맥스가이아 경력에서 팀 자산화 패턴이 면접 필살기와 어떻게 연결되는지 설명해줘"

더 구체적인 질문을 할 수 있게 됐어요. wiki 페이지가 생겼으니까 참조할 수 있는 게 생긴 거예요.

처음엔 Claude가 매번 헤매더라고요. "오늘 무슨 작업이야?" 물어봐야 했거든요. 근데 CLAUDE.md 계층 구조 잡고 나서는 매 세션이 이어지는 느낌이에요. 컨텍스트를 전달하지 않아도 Claude가 Librarian으로 행동해요.
