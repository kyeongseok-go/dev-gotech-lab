안녕하세요, 고텍이예요! 오늘은 좀 특별한 프로젝트 이야기를 들고 왔어요.

운전하시는 분들은 공감하실 텐데요 — **"여기 주차 어디에 해?"** 이 한마디가 매번 스트레스거든요. 네이버 지도 켜서 주차장 검색하고, 요금 비교하고, 리뷰 뒤지고... 매장 자체 주차가 되는지도 확인해야 하고요. 이거 한 번에 해결하는 앱을 직접 만들어보자! 해서 시작한 게 **ParkMate(파크메이트)** 입니다.

결론부터 말하면, **목적지 입력 한 번이면 매장 주차 여부 + 주변 주차장 TOP 3 + AI 요약까지 한 방에** 나옵니다. 같이 고!!

---

## ParkMate가 뭔가요?

한 줄로 말하면 **"스마트 주차 어시스턴트 + 장소 추천 PWA"** 예요.

목적지를 입력하면 이런 일이 벌어져요:

1. 매장 자체 주차 가능한지 확인 (네이버 시설정보 태그 기반)
2. 주변 500m 이내 공영/민영 주차장 TOP 3 추천
3. Claude Haiku가 리뷰를 분석해서 주차 팁을 AI 요약
4. 네이버 지도에서 바로 길찾기까지

이걸로 "강남역 스시코우지" 검색하면, 매장 주차 가능 여부 + 가장 가까운 주차장 3곳 + "이 건물은 지하 2층 주차장이 있고, 2시간 무료입니다" 같은 AI 요약이 한 화면에 뜨는 거죠.

## 개발 여정 — 8단계로 진행했어요

### Phase 1: 프로젝트 초기 셋업

`create-next-app@14`로 시작해서 App Router + TypeScript + Tailwind CSS를 세팅했어요. 여기에 framer-motion, React Query, zustand, shadcn/ui까지 한번에 설치.

특히 신경 쓴 게 **브랜드 색상**이에요. "Forest Guide"라는 팔레트를 만들어서 숲 느낌의 그린 계열(`forest-800: #14352A` ~ `forest-50: #F0FDF4`)로 통일했어요. 주차 가능은 초록, 불가는 빨강, 확인 필요는 노랑 — 신호등처럼 직관적이죠.

PWA 설정도 이때 잡았어요. `manifest.json`, `next-pwa` 연동까지. 나중에 홈 화면에 추가하면 네이티브 앱처럼 돌아가게요.

### Phase 2: 랜딩페이지

framer-motion으로 스크롤 애니메이션을 넣은 히어로 섹션, 기능 소개, 카테고리 그리드, 사용 방법, CTA까지. 첫인상이 중요하니까 여기에 꽤 공들였어요.

### Phase 3: 검색 결과 페이지

이게 핵심이에요. 검색하면 나오는 화면:

- **ParkingCard** — 매장 주차 가능 여부 + AI 요약 텍스트
- **ParkingRankList** — 주변 주차장 TOP 3 (거리, 요금, 주소 표시)
- **NaverMap** — 접이식 네이버 지도에 커스텀 마커 + 인포윈도우

"찾아가기" 버튼을 누르면 Geolocation으로 현재 위치를 잡아서 네이버 지도 앱 `nmap://` 딥링크로 길찾기가 바로 실행돼요.

### Phase 4: FastAPI 백엔드 — 여기서부터 본격 삽질 시작

백엔드가 이 프로젝트의 심장이에요. 외부 API를 **5개**나 연동하거든요:

```
네이버 검색 API (지역) → 매장 시설정보 주차 태그
카카오 로컬 API → 크로스체크 + Geocoding fallback
네이버 Maps Geocoding → 주소/장소명 → 좌표 변환
공공데이터포털 API → 전국 주차장 데이터
Claude Haiku API → 리뷰 분석 + AI 요약
```

이 5개 API의 결과를 하나의 `parking_service.py`에서 오케스트레이션하는 구조예요. 실패 시 fallback 체인도 꼼꼼하게 잡았고요:

```
Redis 캐시 확인 → DB 최근 데이터 → 네이버 API → 카카오 API → 공공데이터만 사용
Claude 호출 → 실패 시 템플릿 기반 텍스트
```

### Phase 5: 데이터 수집

공공데이터포털에서 전국 주차장 데이터를 가져와서 DB에 시드했어요. **13,501건**. PostGIS `ST_MakePoint`로 좌표를 geography 타입으로 저장해서 반경 검색이 가능하게 했어요.

```sql
SELECT * FROM parking_lots
WHERE ST_DWithin(
  location,
  ST_SetSRID(ST_MakePoint(:lng, :lat), 4326)::geography,
  :radius
)
ORDER BY ST_Distance(location, ...)
LIMIT 5;
```

### Phase 6: 장소 추천 기능

데이트/아이와 함께/카페/맛집 같은 카테고리로 장소를 추천해주는 기능이에요. 카카오 키워드 검색 → 네이버 크로스체크 → Claude Haiku 감성 분석 순서로 처리해요.

### Phase 7: 배포

프론트는 Vercel, 백엔드는 Railway로 배포했어요. Supabase PostgreSQL에 PostGIS 활성화하고, Upstash Redis 연결하고, 환경변수 12개 세팅하고... 배포 자체는 이지했는데 환경변수 실수로 좀 고생했어요 ㅋㅋ

### Phase 8: 통합 테스트 & 버그 수정 (현재 진행 중)

여기서 진짜 재밌는(?) 일들이 벌어졌어요. 아래에서 자세히 풀어볼게요.

## 삽질 대방출 — 해결한 6가지 이슈

이 프로젝트에서 제일 값진 경험이 삽질이에요. 제가 다 겪었으니까 여러분은 안 그러셔도 돼요!

### 1. NCloud Geocoding "210 Permission Denied" 💀

이거 제가 반나절 날렸어요...

**증상:** 네이버 Maps Geocoding API 호출하면 무조건 210 Permission Denied.

**삽질 과정:**
- API 키 재발급? → 안 됨
- NCloud 콘솔에서 Maps 구독 확인? → 이용 중으로 뜸
- 결제수단 등록 확인? → 문제 없음

**원인:** 엔드포인트가 바뀐 거였어요!

```python
# 구 엔드포인트 (AI NAVER API — 2025.03 폐지됨)
url = "https://naveropenapi.apigw.ntruss.com/map-geocode/v2/geocode"

# 신 엔드포인트 (Application Services > Maps)
url = "https://maps.apigw.ntruss.com/map-geocode/v2/geocode"
```

NCloud가 "AI NAVER API" 경로를 폐지하고 "Application Services > Maps"로 이전했는데, 기존 문서에 구 URL이 남아있어서 삽질한 거예요. 같은 Client ID/Secret인데 도메인만 달라요.

알고 나면 베리 이지이지~ 근데 모르면 진짜 고생해요.

### 2. Supabase pgbouncer + asyncpg 충돌

Supabase는 pgbouncer를 기본으로 쓰는데, asyncpg의 prepared statement와 충돌해요.

```python
# 해결: NullPool + statement_cache_size 비활성화
engine = create_async_engine(
    settings.database_url,
    poolclass=NullPool,
    connect_args={
        "statement_cache_size": 0,
        "prepared_statement_cache_size": 0,
    },
)
```

`NullPool`은 매 요청마다 새 커넥션을 만들어서 pgbouncer의 transaction mode와 호환돼요.

### 3. SQLAlchemy 버전 호환성

로컬에서는 잘 되는데 Railway 배포하면 크래시! `prepared_statement_cache_size`가 SQLAlchemy 최신 버전에서는 top-level 인자로 안 먹히고 `connect_args`에 넣어야 했어요.

```
TypeError: Invalid argument(s) 'prepared_statement_cache_size'
```

Railway 배포 로그를 보고 잡았는데, 로컬/프로덕션 환경 차이를 늘 조심해야 한다는 교훈을 얻었어요.

### 4. 공공데이터 API — HTTPS가 안 돼요

한국 정부 공공데이터포털 API가 **HTTP만** 지원해요. HTTPS로 호출하면 그냥 차단당합니다.

```python
# HTTPS — 차단됨
url = "https://api.data.go.kr/openapi/tn_pubr_prkplce_info_api"

# HTTP — 정상
url = "http://api.data.go.kr/openapi/tn_pubr_prkplce_info_api"
```

2026년인데... 뭐 어쩔 수 없죠 ㅋㅋ

### 5. asyncpg 타입 호환

asyncpg가 Python `Decimal`과 `datetime.time` 타입을 바로 못 다뤄서 명시적 변환이 필요했어요.

```python
# Decimal → str → float 변환
base_fee = float(Decimal(str(row.base_fee)))
```

### 6. JSON 컨트롤 문자 오류

외부 API 응답에 간혹 컨트롤 문자(`\x00` 등)가 섞여 들어와서 JSON 파싱이 터졌어요. `re.sub`로 정리하는 코드를 추가했어요.

## 핵심 아키텍처 — Geocoding Fallback Chain

이 프로젝트에서 제일 자랑하고 싶은 설계가 **Geocoding Fallback Chain**이에요.

사용자가 "강남역 스시코우지"를 입력하면, 이걸 좌표로 변환해야 주변 주차장을 검색할 수 있잖아요. 근데 하나의 API만 믿으면 장애 때 서비스가 멈춰요.

```
1순위: 네이버 Maps Geocoding API
  ↓ 실패 시
2순위: 카카오 키워드 검색 (좌표 추출)
  ↓ 실패 시
3순위: 네이버 지역검색 mapx/mapy (TM128 좌표 변환)
```

세 겹으로 쌓아서, 하나가 죽어도 나머지가 살아있으면 서비스가 돌아가요. 실제로 네이버 Geocoding이 Permission Denied 뜨던 시점에도 카카오 fallback 덕분에 검색이 가능했어요.

## Claude Haiku AI 연동 — 3종 프롬프트 시스템

AI가 이 서비스의 차별점이에요. Claude Haiku를 3가지 용도로 써요:

**프롬프트 A: 주차정보 분석**
네이버 + 카카오 리뷰를 모아서 "이 매장 주차 어떤가요?"를 JSON으로 응답받아요.

**프롬프트 B: 리뷰 감성 분석**
카테고리별 점수(0.0~1.0)로 장소의 분위기를 수치화해요.

**프롬프트 C: 장소 추천 요약**
존댓말, 이모지 없이, 2~3문장으로 깔끔하게 장소를 소개하는 텍스트를 생성해요.

비용 최적화도 신경 썼어요:
- 리뷰는 **최근 20개만**, 각 **100자 이내**로 잘라서 전달
- 같은 매장 분석 결과는 Redis에 **24시간 캐싱**
- Claude 호출 실패 시 **템플릿 기반 fallback**으로 최소한의 텍스트 제공

## 주차장 검색 — PostGIS 반경 자동 확장

검색 결과가 3개 미만이면 자동으로 반경을 2배로 넓혀요 (최대 3,000m):

```python
radius = request.radius or 500
lots = await search_nearby(lat, lng, radius)

if len(lots) < 3:
    expanded = min(radius * 2, 3000)
    lots = await search_nearby(lat, lng, expanded)
```

교외 지역처럼 주차장이 듬성듬성한 곳에서도 최소 결과를 보장하는 거예요. 쉽죠?
