> 📌 **한 줄 요약:** Cloudflare 풀스택(Pages + R2 + D1 + Workers)으로 서버비 $0/월인 PDF 공유 서비스를 만들었어요.

안녕하세요, 고텍이예요! 오늘은 짧고 굵게 가볼게요.

선생님들이 학부모에게 가정통신문 PDF를 보낼 때, 카카오톡으로 파일을 직접 보내면 **다운로드 만료**되고, 구글 드라이브는 **공유 권한 설정**이 복잡하고, 네이버 MYBOX는 **가입이 필요**하잖아요. "그냥 링크 하나 만들어서 보내면 안 되나?"에서 시작했어요.

결론부터: **가입 없이 드래그 앤 드롭 → 만료되는 공유 링크 즉시 생성 → 서버비 0원**. 이지~하잖아~

## 핵심 플로우

```
PDF 드래그 앤 드롭 → 파일 검증 (MIME + 매직바이트)
  → Cloudflare R2에 저장 → D1에 메타데이터 기록
  → nanoid(8) 기반 공유 URL + QR 코드 생성
  → 카카오톡으로 링크 전송 → 수신자는 가입 없이 열람
  → 만료 시 CRON이 자동 정리
```

## 왜 Cloudflare 풀스택인가

AWS나 Vercel 써도 되는데, 왜 Cloudflare로 올인했냐면요:

| 고민 | Cloudflare 답 |
|------|--------------|
| 파일 저장 비용 | R2 — **이그레스(전송) 비용 $0** |
| DB 비용 | D1 — 서버리스 SQLite, 무료 5GB |
| 서버 비용 | Workers — 무료 10만 req/일 |
| 호스팅 비용 | Pages — 무제한 무료 |

핵심은 **R2의 이그레스 무료**예요. S3는 파일 다운로드할 때마다 전송 비용이 나가는데, R2는 아무리 다운로드해도 0원. PDF 공유 서비스에서 이건 게임 체인저예요.

## 보안 — 교육 문서니까 신경 써야 해요

가정통신문에는 학생 개인정보가 포함될 수 있으니까 보안이 중요해요:

- **이중 파일 검증** — Content-Type + PDF 매직바이트(%PDF, 0x25504446) 확인
- **자동 만료** — 1일/7일/30일/90일 선택, 만료 후 자동 삭제
- **Rate Limiting** — IP당 시간당 10건, 일 50건
- **nanoid(8) URL** — ~2조 가지 조합으로 URL 추측 불가
- **R2 비공개** — 파일은 API를 통해서만 접근, 만료 체크 후 서빙

소프트 삭제 후 30일 뒤 하드 삭제하는 2단계 정리 로직도 넣었어요. 혹시 모르니까요.

## 모바일 최적화 — 카카오톡이 핵심

이 서비스의 사용자 80%는 **카카오톡 인앱 브라우저**로 접근해요. 그래서:

- 카카오톡 인앱 브라우저에서 PDF가 바로 열리도록 pdf.js 렌더링
- iOS Safari의 PDF 인라인 렌더링 처리
- 모바일 핀치 줌 지원 (CSS scale + 디바운스 캔버스 리렌더)
- OG 메타태그로 카카오톡 미리보기 예쁘게

## 결과

**월 운영비 $0**으로 돌아가는 문서 공유 서비스가 탄생했어요. Cloudflare 무료 티어만으로 R2 10GB/월, D1 5GB, Workers 10만 req/일이면 소규모 서비스에 충분하거든요.

만들면서 느낀 건, Cloudflare 풀스택이 **사이드 프로젝트의 최강 조합**이라는 거예요. 서버 관리 없이, 비용 걱정 없이, 글로벌 CDN까지. 복잡하게 생각하지 말고 걍 고!!
