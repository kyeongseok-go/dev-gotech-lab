> 📌 **한 줄 요약:** AI 에이전트 하네스를 진짜 생산성 도구로 만들어주는 Everything Claude Code — 에이전트, 스킬, 훅, 룰 시스템을 실전에서 어떻게 활용하는지 공유해요.

안녕하세요, 고텍이예요! 오늘은 제가 매일 쓰는 AI 개발 환경 이야기를 해볼게요.

요즘 AI로 코딩하는 분들 많잖아요. Claude Code, Cursor, Copilot… 근데 그냥 설치해서 쓰면 **채팅 도구 수준**에 머물러요. "이걸 진짜 개발 플랫폼으로 만들 수 있는 방법이 없을까?" 찾다가 발견한 게 **Everything Claude Code(ECC)**예요.

결론부터: **에이전트 47개 + 스킬 180개 + 이벤트 훅 + 계층형 룰** 시스템을 조합하면, AI 코딩 도구가 완전히 다른 물건이 됩니다. 같이 고!!

## ECC가 뭔가요?

단순한 설정 모음이 아니에요. AI 에이전트 하네스(Claude Code, Cursor, Codex 등)를 **프로덕션 급 개발 플랫폼**으로 만들어주는 시스템이에요. 10개월 이상의 실전 사용에서 진화한 결과물이죠.

핵심 구성 요소 4가지:

| 구성요소 | 개수 | 역할 |
|---------|------|------|
| **Agents** | 47개 | 특화된 서브에이전트 — 코드리뷰, 빌드 에러 해결, 보안 분석 등 |
| **Skills** | 180+개 | 도메인 지식 + 방법론 가이드 — TDD, API 설계, 배포 패턴 등 |
| **Hooks** | 이벤트 기반 | 도구 실행 전후 자동 트리거 — 포맷팅, 린트, 타입체크 등 |
| **Rules** | 계층형 | 항상 따라야 할 가이드라인 — common + 언어별 오버라이드 |

## 에이전트 — 전문가를 호출한다

제가 가장 많이 쓰는 에이전트들이에요:

### 코드 리뷰어 (code-reviewer)
코드를 작성한 직후 바로 호출해요. CRITICAL/HIGH/MEDIUM/LOW 4단계로 이슈를 분류하고, 보안 취약점부터 코드 품질까지 체크해요.

### 빌드 에러 해결사 (build-error-resolver)
빌드가 깨졌을 때 에러 메시지를 분석하고, 최소한의 diff로 빌드를 고쳐줘요. 아키텍처 변경 없이 빌드만 그린으로 — 이게 핵심이에요.

### TDD 가이드 (tdd-guide)
RED → GREEN → REFACTOR 사이클을 강제해요. 테스트 먼저 쓰고, 구현하고, 리팩토링. 커버리지 80% 미만이면 경고.

### 아키텍트 (architect)
복잡한 기능을 설계할 때 시스템 설계, 확장성, 트레이드오프를 분석해줘요.

**핵심은 병렬 실행이에요.** 독립적인 에이전트 3개를 동시에 띄워서 보안 분석, 성능 리뷰, 타입 체크를 병렬로 돌릴 수 있어요. 순차 실행 대비 시간 절약이 엄청나요.

## 스킬 — 도메인 전문성을 주입한다

에이전트가 "누구를 부를까"라면, 스킬은 "어떻게 할까"예요.

제가 직접 만든 스킬 예시:
- **gotekie-blog-writer** — 이 블로그 글을 쓰는 페르소나 가이드! 톤앤매너, 템플릿, 브랜드 연결까지 정의
- **고텍이 카드뉴스** — AI 트렌드를 카드뉴스 형식으로 자동 생성

ECC에 내장된 스킬들:
- **TDD workflow** — 테스트 먼저 쓰는 개발 사이클
- **Security scan** — OWASP Top 10 취약점 스캔
- **E2E runner** — Playwright 기반 E2E 테스트 자동화
- **Continuous learning** — 세션에서 패턴을 추출해서 재사용 가능한 스킬로 변환

## 훅 — 실수를 자동으로 잡는다

이게 진짜 게임 체인저예요. 도구 실행 전후에 자동으로 트리거되는 검증 로직이에요.

### PreToolUse 훅 (실행 전 차단)
- `console.log` 감지 → 경고
- 800줄 초과 파일 작성 차단
- `git push` 전 리마인더

### PostToolUse 훅 (실행 후 자동화)
- 파일 수정 후 **Prettier 자동 포맷**
- `.ts/.tsx` 수정 후 **TypeScript 타입체크**
- 빌드 결과 분석

### 라이프사이클 훅
- **SessionStart** — 이전 세션 컨텍스트 로드
- **SessionEnd** — 세션 상태 저장
- **Stop** — 패턴 추출, 비용 추적

`console.log` 하나 실수로 남겨도 훅이 잡아주니까, 프로덕션에 디버그 코드가 배포되는 사고를 예방할 수 있어요.

## 룰 — 일관성을 강제한다

룰은 **common(공통) + 언어별 오버라이드** 계층 구조예요.

```
rules/
├── common/          # 언어 무관 원칙 (항상 적용)
│   ├── coding-style.md
│   ├── testing.md
│   ├── security.md
│   └── git-workflow.md
├── typescript/      # TypeScript 전용 오버라이드
├── python/          # Python 전용 오버라이드
└── web/             # 웹 프론트엔드 전용
```

예를 들어 `common/coding-style.md`는 "불변성 우선"을 권장하는데, `golang/coding-style.md`에서는 "Go에서는 포인터 리시버로 변경하는 게 관용적"이라고 오버라이드해요. **언어 특성을 존중하면서도 공통 원칙을 유지**하는 구조예요.

## 실제 워크플로우 — 이렇게 씁니다

새 기능을 구현할 때 제 워크플로우예요:

1. **planner 에이전트** → 구현 계획 수립
2. **tdd-guide 스킬** → 테스트 먼저 작성
3. 구현 → **PostToolUse 훅**이 자동 포맷 + 타입체크
4. **code-reviewer 에이전트** → 즉시 코드 리뷰
5. CRITICAL/HIGH 이슈 수정
6. **security-reviewer 에이전트** → 보안 취약점 체크
7. 커밋 → **PreToolUse 훅**이 최종 검증

이 7단계가 거의 자동으로 흘러가요. 제가 직접 하는 건 구현과 판단뿐이고, 나머지 검증은 시스템이 해줘요.

## 직접 만든 스킬 팁

ECC의 진짜 가치는 **커스텀 스킬을 만들 수 있다**는 거예요. `SKILL.md`만 잘 작성하면 AI에게 특정 작업의 전문가 역할을 부여할 수 있어요.

```
~/.claude/skills/my-skill/
├── SKILL.md           # 스킬 정의 (제목, 설명, 트리거)
└── references/        # 참고 자료 (페르소나 가이드, 템플릿 등)
```

블로그 글을 쓸 때마다 톤앤매너가 달라지는 게 싫어서 `gotekie-blog-writer` 스킬을 만들었어요. ESFJ 성격, 브랜드 캐치프레이즈, 콘텐츠 템플릿까지 정의해두니까 **일관된 페르소나로 글이 나와요.**

## 배운 점

- **AI 도구는 설정이 80%** — 기본 설정으로 쓰면 가치의 20%만 누리는 것
- **훅이 가장 ROI 높다** — 실수 방지 자동화는 한 번 설정하면 영원히 돌아감
- **스킬은 지식의 복리** — 한 번 잘 정의하면 프로젝트마다 재사용
- **에이전트 병렬 실행** — 순차보다 병렬이 2~3배 빠름
