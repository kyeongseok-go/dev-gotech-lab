## 1. 실시간 주차 잔여석 — "왜 항상 미확인이지?"

### 문제 발견

ParkMate에서 주차장을 검색하면 "잔여석 미확인"이 계속 떴어요. 분명 배치 수집도 돌고 있고, Redis에 180건이 잘 들어가는데... 왜?

추적해보니까 **근본적인 설계 문제**가 있었어요.

### 두 API의 코드 체계가 달랐다

주차장 데이터를 수집하는 API가 두 개인데:

| API | 용도 | 주차장 코드 |
|-----|------|-----------|
| 공공데이터포털 (data.go.kr) | 전국 주차장 **기본 정보** | `122-2-000007` |
| 서울 열린데이터광장 | 서울 공영주차장 **실시간 잔여석** | `1037932` |

이 두 코드가 **완전히 다른 체계**였어요. DB에는 공공데이터 코드(`122-2-000007`)가 `public_id`로 들어있는데, 실시간 배치가 서울 코드(`1037932`)로 매칭을 시도하니까 당연히 0건 매칭...

```python
# collect_realtime.py — 이 쿼리가 항상 0건 반환
result = await session.execute(
    text("SELECT id FROM parking_lots WHERE public_id = :pid"),
    {"pid": parking_code},  # 서울 API의 PKLT_CD
)
# parking_lots.public_id에는 공공데이터의 prkplceNo가 들어있으니 매칭 불가!
```

이거 발견했을 때 진짜... ㅋㅋ 이름 매칭도 시도해봤는데, DB에는 `강남치매안심센터`, `언북초교` 같은 부지 이름이, 서울 API에는 `세종로 공영주차장(시)` 같은 정식 명칭이 들어있어서 이름으로도 매칭이 안 됐어요.

### B안 — 서울 API 데이터를 DB에 직접 동기화

A안(억지로 매칭)과 B안(서울 API 데이터를 DB에 넣기) 중에서 **B안**을 선택했어요.

핵심 아이디어: 서울 API 주차장을 `parking_lots`에 직접 UPSERT하면, `public_id = PKLT_CD`로 저장되니까 Redis 실시간 데이터와 바로 매칭됩니다.

```
[1회성/주간 배치]
서울 API → 주소를 네이버 Geocoding → 좌표 확보 → parking_lots UPSERT

[5분 간격 배치]
서울 API → Redis 저장 (키: realtime:parking:{PKLT_CD})

[검색 시점]
PostGIS 반경 검색 → public_id로 Redis 조회 → 잔여석 매칭!
```

`collect_seoul_parking.py`를 새로 만들어서 실행했더니 **179건 UPSERT 성공**. 그리고 실시간 매칭 테스트:

```
✅ 세종로 공영주차장(시) → 잔여 740/1260
✅ 종묘주차장 공영주차장(시) → 잔여 186/1317
✅ 한강진역 공영주차장(시) → 잔여 81/174
```

10/10 완벽 매칭! 해보니까 이지~하잖아~ (삽질 3시간은 비밀)

### 아키텍처 최종 구조

`collect_realtime.py`도 DB FK 의존을 완전히 제거하고 Redis 전용으로 단순화했어요:

```python
# Before: DB에 INSERT (매칭 실패로 0건)
# After: Redis만 저장 + 이름 인덱스
pipe.set(key, json.dumps(data), ex=600)  # 10분 TTL
pipe.hset("realtime:parking:name_index", normalized_name, parking_code)
```

서버 시작 시 `collect_seoul_parking`이 자동 실행되도록 lifespan에도 추가했어요.

## 2. 전국으로 확장 — API 4곳 신청

서울만으로는 부족하죠. 전국 실시간 잔여석을 위해 API를 조사하고 신청했어요:

| API | 상태 | 특징 |
|-----|------|------|
| 한국교통안전공단 (전국) | 승인 대기 | 기존 `prkplceNo`와 동일 코드 체계! |
| 대구 통합주차정보 | 승인 대기 | 혼잡도 + 잔여석 |
| 경기도 교통정보센터 | 키 발급 완료 | `avblPklotCnt` 필드 |
| 광주 C-ITS | 교통안전공단으로 커버 예정 | |

교통안전공단 API가 승인되면 전국이 한 번에 커버되는데, 기존 `parking_lots.public_id`와 코드 체계가 같아서 **코드 변경 없이 바로 매칭**될 거예요.

## 3. 카카오톡 공유 — 바이럴의 핵심

"토요일에 성수 OO 가자" → "거기 주차 어떻게 해?" → **ParkMate 검색 결과 공유**

이 시나리오가 핵심이라서 공유 기능을 구현했어요.

### /share 페이지 — 동적 OG 태그

공유 링크를 카카오톡에 보내면 예쁜 미리보기가 떠야 하잖아요. 근데 `/search` 페이지는 클라이언트 렌더링이라 OG 태그가 동적으로 안 바뀌어요.

그래서 **`/share?q=강남역+스시코우지`** 공유 전용 페이지를 서버 컴포넌트로 만들었어요:

```typescript
// 서버에서 백엔드 API 호출 → OG 태그에 검색 결과 요약 삽입
export async function generateMetadata({ searchParams }: SharePageProps): Promise<Metadata> {
  const data = await fetchParkingData(q);
  return {
    title: `${data.place.name} 주차 정보`,
    description: buildDescription(data),  // "자체주차 불가 | 주변 주차장 3곳 | 1위: 강남 공영 2분"
  };
}
```

크롤러(카카오톡 스크래퍼)는 메타데이터만 읽고, 실제 사용자는 1.5초 후 `/search`로 리다이렉트됩니다.

### 공유 드롭다운 메뉴

검색 결과에 공유 버튼을 추가하고, 3가지 공유 방법을 넣었어요:

- **공유하기** — Web Share API (모바일 기본 공유 시트)
- **카카오톡 공유** — Kakao JS SDK feed 타입 카드
- **링크 복사** — Clipboard API + fallback

카카오 JS SDK 키 발급부터 Redirect URI 설정까지 한 번에 진행했어요. 쉽죠?

## 4. SEO — 검색엔진에 노출되자

서비스를 만들어놓고 검색이 안 되면 의미가 없으니까요.

### sitemap.ts + robots.ts

Next.js 14 App Router에서는 타입이 지정된 함수로 sitemap과 robots를 생성할 수 있어요:

```typescript
// sitemap.ts
export default function sitemap(): MetadataRoute.Sitemap {
  return [
    { url: BASE_URL, changeFrequency: "daily", priority: 1 },
    { url: `${BASE_URL}/search`, changeFrequency: "always", priority: 0.9 },
    { url: `${BASE_URL}/trending`, changeFrequency: "daily", priority: 0.8 },
    // 카테고리별 페이지...
  ];
}
```

### 페이지별 고유 메타데이터

이게 중요한 포인트인데요, 모든 페이지가 같은 title/description이면 검색엔진이 구분을 못 해요. 그래서 각 페이지마다 SEO에 맞는 메타데이터를 넣었어요:

| 페이지 | 검색 결과 |
|--------|----------|
| `/trending` | **트렌드 장소 \| ParkMate** — 성수, 홍대, 강남 요즘 뜨는 핫플레이스 |
| `/category/cafe` | **주차 편한 카페 추천 \| ParkMate** — 주차 걱정 없는 신상 카페 |
| `/category/food` | **주차 편한 맛집 추천 \| ParkMate** — 주차하기 좋은 맛집 |

누군가 네이버에서 "주차 편한 카페"를 검색하면 ParkMate 카테고리 페이지가 잡힐 수 있는 거죠.

Google Search Console, Naver Search Advisor에도 등록하고 sitemap을 제출했어요.

## 5. 주차 요금 계산기 — 생각보다 복잡했다

"2시간 주차하면 얼마야?" 를 자동 계산하는 기능인데, 실제 데이터를 보니까 과금 케이스가 다양했어요.

### 발견한 과금 패턴

```
조선호텔앞: 기본 5분/0원, 추가 5분/600원  → 5분 무료 후 5분당 600원
관철동(시): 기본 5분/430원, 추가 5분/430원  → 5분당 430원 균일
관철동:     기본 5분/430원, 추가 0분/0원    → 추가 데이터 없음 (옛 데이터)
무료:       기본 0분/0원, 추가 0분/0원      → 완전 무료
```

처음에는 단순하게 `기본요금 + (초과시간/추가단위) × 추가요금`으로 했는데, "5분 0원"인 주차장이 2시간에 13,800원으로 표시되는 건 맞지만 "5분 0원"이라고 써놓으면 무료처럼 보이는 문제가 있었거든요.

### 4가지 케이스 대응

```typescript
// 케이스 1: 기본 무료 + 추가 과금 → "5분 무료 / 이후 5분 600원"
// 케이스 2: 균일 과금 → "5분당 430원"
// 케이스 3: 추가 데이터 없지만 단위가 짧으면 단위 과금으로 간주
// 케이스 4: 일반 기본+추가 구조
```

시간 선택도 토글로 30분/1시간/2시간/3시간 중에 고를 수 있게 했고, 주말/야간 요금 차이 안내도 추가했어요.

## 6. 네이버 지도 — 두 가지 수정

### 렌더링 깨짐 해결

지도가 접힘→펼침 전환될 때 타일이 안 나오는 이슈가 있었어요. CSS transition 중에 지도 크기가 아직 0일 때 초기화되면서 생기는 문제였는데, `autoResize()`를 50ms/350ms/700ms 3회 반복 호출하는 것으로 해결했어요.

### 검색 장소 마커 추가

기존에는 주차장 마커(초록 1,2,3)만 있었는데, 검색한 장소(목적지)가 어디인지 모르니까 불편했거든요. 빨간 마커를 추가하고, fitBounds로 모든 마커가 화면에 보이도록 자동 줌도 넣었어요:

```
📍 빨간 마커 — 검색한 장소 (목적지)
🟢 1, 2, 3 — 추천 주차장
```

## 7. 전기차 충전소 — 한국환경공단 API

전기차 운전자를 위해 주변 충전소 정보도 보여주기로 했어요. 한국환경공단 API를 신청하고 연동했는데, 이 API는 위치 기반 검색을 지원하지 않아서 **시/도 단위(zcode)로 호출 → haversine 거리 계산으로 반경 필터**하는 구조로 만들었어요.

Redis 30분 캐싱으로 일 1,000건 제한도 문제없고요. 검색 결과에 이렇게 표시돼요:

```
⚡ 주변 전기차 충전소
  세종로 공영 충전소  [급속] [완속] [주차무료]
  📍 230m · 충전기 4대 · 2대 사용 가능
```

## 8. UX 개선 모음

### 로그인 안심 문구

모르는 사이트에 카카오/네이버로 로그인하면 거부감이 들 수 있잖아요. 그래서:

- 로그인 페이지에 🔒 **"안심 로그인 — 닉네임과 프로필 이미지만 사용"** 배너
- **"로그인은 선택이에요"** — 검색, 트렌드, 추천은 로그인 없이 이용 가능
- 헤더에 최초 1회 말풍선 툴팁
- 즐겨찾기 페이지에서도 보안 안내

### 브랜드 로딩 애니메이션

스켈레톤 로딩 대신, ParkMate 로고가 SVG로 그려지는 애니메이션을 넣었어요. 검색 로딩은 초록색, 즐겨찾기 로딩은 골드색으로 각각 다른 테마로!

### 인기 검색어 개선

"강남역 스시코우지" 같은 하드코딩을 제거하고, 실제 검색 로그에서 인기 검색어를 가져오되 부족하면 트렌드 장소(FIRE/HOT/RISING)로 보충하도록 바꿨어요. 트렌드 데이터도 90일 이내만 표시되도록 필터를 추가했고요.

## 오늘의 삽질 포인트

여러분이 비슷한 상황에 처하면 이것만 기억하세요:

1. **공공데이터 API끼리 코드 체계가 다를 수 있다** — 같은 주차장인데 다른 번호를 쓸 수 있어요. 연동 전에 반드시 샘플 데이터를 비교하세요.

2. **data.go.kr 활용 신청은 서비스별로 따로 해야 한다** — 같은 API 키라도 서비스별 권한이 다릅니다. 승인 반영도 시간이 걸려요.

3. **네이버 지도 resize 이슈** — CSS transition으로 높이를 바꿀 때 `autoResize()`를 한 번만 호출하면 안 되고, transition 진행 중에 여러 번 호출해야 해요.

4. **주차 요금 데이터가 균일하지 않다** — "기본 5분 0원"이 무료가 아니라 "5분 무료 후 유료"일 수 있어요. 케이스별 분기 필수.
