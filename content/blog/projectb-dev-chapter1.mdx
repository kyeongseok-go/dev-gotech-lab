## 프로젝트는 어떻게 시작됐나

오프라인 핸드메이드 악세사리 매장을 운영하시는 대표님에게서 연락이 왔어요. "온라인 쇼핑몰 만들어주세요." 근데 예산은... 100만원 이하 ㅋㅋ 실제 필요한 금액은 700~1,050만원이거든요. 갭이 크죠?

그래서 결심했습니다. **AI 협업 1인 개발**로 해보자고!

### 핵심 요구사항 정리

대표님과 미팅하면서 체크리스트를 돌렸는데요. 주요 포인트만 뽑으면:

- **오프라인 매장이 메인** → Store Location 페이지가 핵심
- **상품 10~50개** → 단일 품목, 퀄리티 강조
- **비개발자가 직접 관리** → 관리자 대시보드 필수
- **결제**: 카드, 카카오페이, 네이버페이 등 다양하게
- **재구매율 50% 이상** → 회원 관리, 포인트/쿠폰 중요

## 기술 스택 선정

고민할 것 없이 바로 정했어요. 쉽죠?

```
Frontend:  Next.js 14 + TypeScript + Tailwind CSS (App Router)
Backend:   Supabase (PostgreSQL + Auth)
Images:    Cloudflare R2
Deploy:    Vercel
Payment:   포트원 (PortOne V2)
```

Next.js App Router를 선택한 이유는 **서버 컴포넌트**로 Supabase 쿼리를 바로 날릴 수 있어서예요. API Route 따로 안 만들어도 되는 케이스가 많거든요.

Tailwind CSS는... 이건 그냥 국룰이니까 넘어갈게요 ㅎㅎ

## 디자인 시스템 — Industrial Minimal

이 프로젝트의 디자인 핵심은 **"Generic AI aesthetic 절대 금지"**예요.

Inter 폰트? ❌ 보라색 그라데이션? ❌ 둥근 모서리? ❌❌❌

모든 곳에 `border-radius: 0`. 각지고, 모노크롬이고, 여백을 최대한 살리는 Industrial Minimal 스타일이에요.

### 컬러 팔레트

```
Jet Black    #0A0A0A  ← 기본 전경
Rich Black   #1A1A1A  ← 어두운 표면
Charcoal     #333333  ← 보조 텍스트
Silver       #999999  ← 비활성/힌트
Light Gray   #D4D4D4  ← 테두리
Snow         #FAFAFA  ← 배경

Accent:
Sale Red     #C75050  ← 세일, 에러
```

진짜 흑백이죠? 근데 이게 핸드메이드 소품에 딱 맞아요. 상품 사진이 돋보이거든요.

### 타이포그래피

- **Heading**: Archivo — 기하학적 산세리프, UPPERCASE + 넓은 자간
- **Body**: Pretendard Variable — 한국어 최적화 산세리프

이 조합이 "공방 갤러리" 느낌을 잘 살려주더라고요.

## 18개 페이지 UI 완성

Claude Code한테 CLAUDE.md에 디자인 시스템과 와이어프레임을 상세하게 적어놓고 작업했어요. 이게 핵심이에요 — **컨텍스트를 잘 줘야 결과물이 좋다**.

### 완성된 페이지 목록

| 페이지 | 설명 |
|--------|------|
| Home | 히어로 슬라이드쇼 + New In/Best Sellers 그리드 + 뉴스레터 |
| Category | 태그 필터(handmade/fabric/metal 등) + 페이지네이션 |
| Product Detail | 이미지 갤러리 + 옵션 선택 + 리뷰 + 아코디언 |
| Cart | 수량 조절 + 20개/페이지 페이지네이션 |
| Checkout | 배송 정보 + 결제 수단 선택 + 주문 요약 |
| Auth | 소셜 로그인 (카카오/네이버/구글) + 이메일 |
| MyPage | 주문내역/위시리스트/프로필/포인트 탭 |
| Brand Story | 브랜드 스토리 + 소재 소개 3칸 그리드 |
| Store Location | 지도 + 매장 정보 + 길찾기/카카오톡 CTA |
| Magazine | 블로그/매거진 + 대표 글 |
| Photo Reviews | 메이슨리 그리드 리뷰 |
| CS Center | FAQ/Q&A/기타문의/도매제휴/공지 5탭 |
| Notice | 공지/이벤트 아코디언 |
| Terms/Privacy | 약관/개인정보처리방침 |
| Checkout Success/Fail | 결제 결과 페이지 |

18개 페이지를 더미 데이터로 전부 동작하게 만들었어요. Zustand로 장바구니, 위시리스트, 최근 본 상품까지 로컬 상태 관리 완료.

### 삽질 포인트 🔥

**4/2 대표님 미팅 후 변경사항이 꽤 있었어요:**

1. "Shop" → "Category"로 네비/라우트 이름 변경
2. REVIEWS 별도 페이지 → 상품 상세페이지 안으로 이동
3. 문의 탭을 5개로 재구성 (FAQ/Q&A/기타/도매·제휴/공지)
4. 장바구니 20개/페이지 페이지네이션 추가

대표님이 **비개발자**시라 직접 시연하면서 피드백을 받았는데, 실제 사용자 관점의 피드백이라 엄청 유용했어요. "리뷰를 따로 보러 가기 귀찮다" → 상세페이지에 넣자! 이런 거죠.
