## 문제 인식 — 주차 정보가 너무 흩어져 있다

주차 정보를 얻으려면 보통 이런 과정을 거치잖아요:

1. 네이버 지도에서 목적지 검색 → 주차 가능 여부 확인 (태그가 있으면 운이 좋은 거)
2. 주변 공영주차장 별도 검색
3. 리뷰에서 "주차" 키워드로 직접 필터링
4. 실시간 잔여석? 서울시 앱 따로 확인

**4개 앱/사이트를 돌아다녀야 하는 거예요.** 이걸 한 화면에 모으면 되겠다 싶었어요.

## 기술 스택 선택

### 프론트엔드: Next.js 14 + Tailwind CSS

모바일 퍼스트 PWA가 목표였기 때문에 App Router 기반으로 가되, 지도가 핵심이니까 클라이언트 컴포넌트를 적극 활용했어요. UI 컴포넌트는 shadcn/ui, 애니메이션은 Framer Motion으로 가볍게 처리했어요.

### 백엔드: FastAPI + PostgreSQL + Redis

Python 생태계의 지도/위치 라이브러리가 풍부하고, FastAPI의 async 지원이 5개 외부 API를 동시 호출하기에 딱이었어요. 데이터베이스는 PostGIS 확장이 필요하니까 PostgreSQL은 필수였고, Redis는 7단계 캐시 전략의 핵심이에요.

### AI: Claude Haiku

리뷰 분석에 GPT-4를 쓰기엔 비용이 부담됐거든요. Claude Haiku는 호출당 약 700토큰(입력 500 + 출력 200)으로, DAU 1,000 기준 **월 3~5천원** 수준이에요. 가성비 끝판왕이죠.

## 5개 API 무료 티어 연동기

이 프로젝트의 가장 큰 도전은 **모든 API를 무료 티어로 운영하는 것**이었어요.

| API | 무료 한도 | 역할 |
|-----|----------|------|
| 네이버 검색 API | 25,000 req/일 | 주차 태그 소스 (가장 신뢰도 높음) |
| 네이버 지도 API | 600만 req/월 | 지도 렌더링 + Geocoding |
| 카카오 로컬 API | 100,000 req/일 | 교차 검증 |
| 공공데이터포털 | 10,000 req/일 | 전국 주차장 DB |
| 서울열린데이터 | 무제한 | 실시간 잔여석 |

핵심은 **폴백 체인**이에요:

```
1. Redis 캐시 확인
2. DB 히스토리 데이터
3. 네이버 API → 카카오 API → 공공데이터만
4. Claude 호출 → 템플릿 기반 텍스트 생성
```

하나가 죽어도 다른 소스로 대체되니까, 사용자는 항상 결과를 받을 수 있어요.

## Claude Haiku 리뷰 분석

가장 재밌었던 부분이에요! 네이버와 카카오 리뷰에서 주차 관련 멘션을 뽑아서 Claude Haiku에게 분석을 맡겨요.

**최적화 포인트:**
- 최근 리뷰 20개만, 각 100자 제한으로 입력 토큰 절약
- 감성 분석 점수(0.0~1.0)로 주차 난이도를 수치화
- 분석 결과는 **24시간 캐시** — 같은 장소 재검색 시 AI 재호출 없음

"주차 넓어요", "발렛 있음", "주말에 주차 전쟁" 같은 리뷰들을 Claude가 읽고 **"주차 공간이 넉넉하며 발렛 서비스 이용 가능. 다만 주말 오후에는 혼잡할 수 있어요."** 같은 자연어 요약을 만들어줘요. 이지~하잖아~

## PostGIS 반경 검색

목적지 좌표 기준 500m 반경 내 공영주차장을 찾는 건 PostGIS가 딱이에요.

```sql
CREATE INDEX idx_parking_lots_location ON parking_lots USING GIST (
  ST_SetSRID(ST_MakePoint(longitude, latitude), 4326)
);
```

GIST 인덱스 하나로 수만 개 주차장 중에서 반경 내 결과를 밀리초 단위로 뽑아내요. 거기에 거리/비용/잔여석을 조합한 **복합 랭킹**으로 TOP 3를 선별하죠.

## 7단계 Redis 캐시 전략

외부 API 호출을 최소화하면서 데이터 신선도를 유지하는 게 관건이었어요.

| 캐시 키 | TTL | 이유 |
|---------|-----|------|
| 검색 결과 | 1시간 | 같은 검색어 반복 방지 |
| 실시간 잔여석 | 10분 | 서울시 API 갱신 주기에 맞춤 |
| 장소 상세 | 24시간 | 가게 정보는 자주 안 바뀜 |
| 감성 분석 | 7일 | AI 호출 비용 절감 핵심 |
| 카테고리 추천 | 6시간 | 트렌드 반영과 비용 사이 균형 |
| 트렌드 | 24시간 | 일별 갱신이면 충분 |

## PWA + 배포

모바일에서 주로 쓸 앱이니까 PWA는 필수였어요.

- 홈 화면 설치 지원 (192×192, 512×512 아이콘)
- 오프라인 캐싱으로 최근 검색 결과 유지
- 테마 컬러: Forest Guide 팔레트 (#14352A 기반)

배포는 프론트엔드 Vercel(무료), 백엔드 Railway(약 7,000원/월), DB Supabase(무료), Redis Upstash(무료). **총 운영비 월 7,000원 수준**이에요!

## 배운 점

- **무료 API 조합의 위력** — 5개 API를 잘 엮으면 유료 서비스 못지않은 데이터를 모을 수 있어요
- **AI 비용은 캐시로 잡는다** — Claude Haiku + 적극적 캐싱으로 AI 비용을 거의 무시할 수 있는 수준으로
- **폴백 체인은 필수** — 외부 API 의존 서비스에서 하나라도 죽으면 전체가 죽는 건 치명적
- **PostGIS는 위치 기반 서비스의 치트키** — 순수 SQL로 반경 검색하려면 고통받는데, PostGIS면 이지~
