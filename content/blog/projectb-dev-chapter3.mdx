## Phase 3 — 콘텐츠 편집

대표님이 4/2 미팅에서 가장 강하게 원했던 기능이에요. "히어로 이미지를 내가 직접 바꿀 수 있어야 해요!"

### 히어로 섹션 편집

관리자 페이지에서 수정할 수 있는 항목:
- **텍스트**: 메인 타이틀, 서브 타이틀
- **CTA 버튼**: 텍스트 + 링크 (2개)
- **슬라이드 이미지**: 추가/삭제/순서 변경

DB 구조가 깔끔해요:
- `pb_hero_settings` — 텍스트/CTA (단일 row)
- `pb_hero_slides` — 슬라이드 이미지 (복수 rows)

프론트엔드도 전환했어요. 기존에 `constants/home.ts`에 하드코딩되어 있던 `HERO_CONTENT`와 `HERO_SLIDES`를 DB에서 조회하도록 변경.

```typescript
// 기존: 하드코딩
import { HERO_CONTENT, HERO_SLIDES } from "@/constants/home";

// 변경: DB 조회
const heroData = await getHeroData(); // Supabase 쿼리
<HeroSection data={heroData} />
```

### 브랜드 페이지 편집

브랜드 페이지는 5개 섹션으로 나뉘어요:
- `hero` — 브랜드 페이지 상단
- `story` — 브랜드 스토리
- `material_1` ~ `material_3` — 소재 소개 3칸

각 섹션마다 제목, 내용(textarea), 이미지를 수정할 수 있어요. `section_key`로 구분하고, upsert으로 저장.

## Phase 4 — 주문/CS 관리

### 주문 관리

주문 목록 페이지에 상태 필터 탭을 넣었어요:

```
[전체(5)] [결제완료(2)] [준비중(1)] [배송중(1)] [배송완료(1)] [취소(0)]
```

주문 상세에서는:
- 주문 정보 (번호, 고객, 연락처, 이메일, 결제 방법)
- 주문 상품 목록 (이름, 옵션, 수량, 가격)
- **상태 변경** (드롭다운으로 pending → paid → preparing → shipped → delivered)
- **배송 추적번호 입력**

대표님이 엑셀로 관리하시던 걸 웹에서 할 수 있게 된 거예요!

### 리뷰/문의/공지 관리 (진행 중)

이 부분은 아직 완성 중이에요:

- **리뷰 관리**: 리뷰 목록 + 관리자 답변 작성 + 공개/비공개 토글
- **문의 관리**: 문의 목록 (상태 필터) + 답변 작성
- **공지사항**: CRUD (생성/수정/삭제 + 고정/공개 토글)
