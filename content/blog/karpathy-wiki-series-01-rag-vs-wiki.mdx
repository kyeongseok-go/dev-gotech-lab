## 이직 준비하면서 Notion이 터진 이야기

2024년 12월에 티맥스가이아에서 퇴사했어요. 5년 5개월 재직했고, 오피스 SW 엔진에서 텍스트 엔진, 행정 인프라 구축, PDF 렌더링 최적화 같은 작업들을 했어요.

근데 퇴사하고 이직 준비를 시작하니까 문제가 생겼어요.

1년 안에 안정적인 대기업에 들어가야 한다는 압박이 있는 상태에서, 자소서 쓰고, 코딩 테스트 준비하고, 포트폴리오 AI 프로젝트 만들고, 회사별 면접 준비까지 해야 하잖아요. 그 모든 걸 Notion 한 페이지에 때려박았더니 어느 순간부터 그 페이지 열기가 두려워졌어요 ㅋㅋ

"그때 한화에어로스페이스 자소서에서 뭐라고 썼더라?" 찾으면 5분.
"LG CNS 면접 핵심 키워드가 뭐였지?" 또 스크롤 3분.
Claude한테 "지난번에 정리한 내 경력 강점이 뭐였어?" 물어봐도 "이전 대화 내용은 기억하지 못해요."

컨텍스트가 없으니까 매번 처음부터 설명해야 했어요. 그게 정말 비효율적이더라고요.

## RAG를 공부해봤는데

이 문제 해결하려고 RAG(Retrieval-Augmented Generation)를 공부했어요.

RAG는 문서를 작은 청크로 잘라서 벡터로 임베딩하고, 질문이 들어오면 유사도 높은 청크를 검색해서 LLM에 컨텍스트로 주는 방식이에요. 검색 엔진 + LLM 조합이라고 보면 돼요.

근데 제 상황에서 RAG를 쓰려면:

1. 임베딩 파이프라인 구축 (pgvector 세팅, 임베딩 모델 선택)
2. 문서 청킹 전략 설계
3. 검색 API 개발
4. LLM 연동

이걸 1인이 이직 준비하면서 동시에? 솔직히 현실적이지 않았어요.

그리고 더 근본적인 문제가 있었어요. **RAG는 검색하는 시스템이에요. 합성하는 시스템이 아니에요.**

"내가 한화 면접에서 어필해야 할 포인트가 LG CNS 면접 전략과 어떻게 달라야 하나?" 같은 질문엔 RAG가 약해요. 서로 다른 문서에서 청크를 가져와도, 그걸 **합성해서 재해석**해주진 않거든요.

## Karpathy 글에서 발견한 것

오늘 읽은 Karpathy 글의 핵심 아이디어:

> LLM을 "검색기"로 쓰지 말고 "컴파일러"로 써라. 소스 자료를 축적하면서 그것을 영구적인 wiki 지식으로 컴파일한다.

RAG가 검색할 때마다 임베딩 공간을 뒤지는 거라면, LLM Wiki는 소스를 인제스트할 때마다 wiki 페이지를 새로 쓰거나 갱신해요. 마치 컴파일러가 소스를 받아 점점 더 완성되는 바이너리를 만드는 것처럼요.

비교해보면 이래요:

| | RAG | LLM Wiki |
|---|---|---|
| 핵심 동작 | 청크 검색 | wiki 컴파일 |
| 질문 응답 | 유사 청크 조합 | 재해석된 합성 |
| 지식 형태 | 분산 벡터 | 연결된 그래프 |
| 유지보수 | 임베딩 재인덱싱 | wiki 페이지 업데이트 |
| 구축 비용 | 벡터 DB + 파이프라인 | Obsidian + Claude Code |

## 왜 1인 개발자에게 특히 잘 맞는가

세 가지 이유가 있어요.

**첫째, 인프라 없이 시작할 수 있어요.**
벡터 DB 없어도 돼요. Obsidian + Claude Code + GitHub만 있으면 오늘 당장 시작할 수 있어요. 저도 오늘 막 시작했거든요 ㅋㅋ 베리 이지이지~

**둘째, 지식이 합성되면서 쌓여요.**
RAG는 문서를 저장하지만, LLM Wiki는 그 문서를 해석해서 재사용 가능한 형태로 변환해요. 한화 채용 프로세스를 ingest하면 "서류 → PT면접 → 임원면접 → 처우" 같은 패턴이 wiki 페이지가 되고, 그게 나중에 다른 회사 면접 준비할 때도 재활용되는 거예요.

**셋째, Claude 컨텍스트 문제가 해결돼요.**
Claude가 이전 대화를 기억 못 한다고 했잖아요? 근데 wiki를 읽게 하면 컨텍스트가 생겨요. "내 경력의 강점이 뭐야?"라고 물으면 wiki에서 읽고 답해줘요. 매 세션마다 처음부터 설명하는 비효율이 없어지는 거예요.

## 오늘 시작한 것

지금 이 글을 쓰는 동안 Obsidian에 Gotechy-Wiki라는 vault를 막 만들었어요.

구조는 3-layer로 잡았어요:

```
raw/     ← 원본 소스 불변 보관
wiki/    ← AI가 컴파일한 지식 그래프
Output/  ← 자소서, 블로그, 발표자료
```

CLAUDE.md도 오늘 처음 잡았어요. Claude한테 "이 vault에서 어떻게 일해야 해"를 규칙으로 써두는 파일이에요. 자세한 건 2편에서 다룰게요.

솔직히 아직 wiki 페이지가 거의 없어요. 오늘 막 시작했으니까요. 근데 이게 맞는 방향인 것 같아서 흥분 가시기 전에 먼저 썼어요.
