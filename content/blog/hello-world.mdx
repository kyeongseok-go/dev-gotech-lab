안녕하세요, 고텍이예요! 오늘도 쉽게 가보자고!

드디어 개인 블로그를 시작합니다. **gotech.lab** — 이름에서 느껴지시죠? Go(고경석) + Tech(기술) + Lab(실험실). 기술을 직접 만들어보고, 그 과정을 솔직하게 나누는 공간이에요.

---

## 왜 시작했나

5년간 오피스 SW 엔진을 만들면서 핵심 모듈 9개를 설계했어요. TypeScript, C++, 문서 렌더링 엔진... 깊이 있는 작업을 해왔는데, 정작 이 경험을 밖으로 꺼내본 적이 없더라고요.

요즘은 AI와 함께 1인 개발도 하고 있는데, 이 과정이 너무 재밌어서 기록하고 공유하고 싶었어요. 누군가에게는 "아, 이렇게 하면 되는구나" 하는 힌트가 될 수 있을 것 같아서요.

---

## 어떤 이야기를 하나요

크게 세 가지 주제를 다룰 거예요:

### 1. AI와 함께하는 개발 이야기
Claude Code로 프로젝트를 처음부터 끝까지 짓는 경험, AI 도구 리뷰, 프롬프트 엔지니어링 팁 같은 것들이요. 직접 써보고 솔직하게 풀어볼게요.

### 2. 프로젝트 개발기
ParkMate(스마트 주차 앱), Smart Thumbnail Generator, Markdown to Slides 같은 프로젝트들을 만들면서 겪은 삽질과 해결 과정을 공유해요. 삽질 포인트를 미리 알려드릴게요 — 여러분은 안 그러셔도 되게요!

### 3. 웹 기술 딥다이브
Next.js, TypeScript, Tailwind, Cloudflare Workers 같은 기술들을 실전에서 어떻게 쓰는지 경험 기반으로 정리해요. 공식 문서 번역이 아니라 "직접 써보니 이랬다"는 이야기요.

---

## 기술 스택

이 블로그 자체도 직접 만들었어요:

| 기술 | 용도 |
|------|------|
| **Next.js 16** App Router | 프레임워크 |
| **Tailwind CSS v4** + shadcn/ui | 스타일링 |
| **Velite** + MDX | 콘텐츠 파이프라인 |
| **Cloudflare Pages** | 배포 (엣지 성능 + 무료!) |

<Callout type="info">
이 블로그는 MDX 기반이라 코드 블록, 커스텀 컴포넌트, 테이블 등을 자유롭게 쓸 수 있어요.
</Callout>

<Callout type="tip">
`Callout` 컴포넌트는 info, warning, tip 세 가지 타입을 지원해요. 중요한 내용을 강조할 때 쓸게요!
</Callout>
