## 프로젝트 소개

항공·방산 R&D 현장에서는 **ECO(Engineering Change Order)** 가 발생하면 개발자가 직접 영향 분석부터 코드 수정, 테스트 케이스 작성, 변경 보고서 작성까지 수작업으로 처리합니다.

**ECO-DevAssist**는 이 4가지 작업을 Claude Sonnet이 자동 수행하도록 만든 프로토타입입니다. BOM(자재명세서) 트리 기반으로 형상변경 영향을 시각화하고, 6탭 UI에서 전체 워크플로우를 한 화면에서 처리할 수 있습니다.

## 6탭 워크플로우

| 탭 | 기능 |
|---|---|
| ECO 대시보드 | ECO 목록·통계·우선순위 관리 |
| 영향 분석 | AI가 연관 SW 모듈 및 리스크 자동 분석 |
| 코드 생성 | Before/After diff 형태로 AI 코드 수정안 제시 |
| 테스트 생성 | 변경 모듈 대상 테스트 케이스 자동 생성 |
| 보고서 생성 | 형상변경 보고서 다운로드 |
| 지식 검색 | 자연어로 과거 ECO 이력 검색 |

## 기술 하이라이트

- **BOM 트리 시각화** — K-FCS 화력통제시스템 → 서브시스템 → 조립체 계층 구조
- **Claude Sonnet 4종 프롬프트** — 영향 분석 / 코드 생성 / 테스트 생성 / 보고서 생성 특화
- **Mock 데이터** — SW 모듈 11개, ECO 3개, 과거 이력 5개로 실제 R&D 시나리오 재현
- **useReducer 기반 상태 관리** — 탭 간 ECO 선택 컨텍스트 공유
