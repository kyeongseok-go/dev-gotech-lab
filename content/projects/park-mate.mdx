## 프로젝트 소개

"여기 주차 되나?" 한 번의 검색으로 인근 주차장 유무, 실시간 잔여석, AI 리뷰 분석, 네이버 지도 딥링크까지 한 화면에 보여주는 PWA입니다.

주차 정보가 네이버 지도, 카카오 맵, 공공데이터, 실시간 잔여석 서비스에 흩어져 있다는 문제에서 시작했습니다.

## 10단계 검색 파이프라인

```
Geocoding (3-레이어 폴백: 네이버 → 카카오 → 지역검색)
  → 네이버 지역검색 (1순위)
  → 카카오 크로스체크 (2순위)
  → PostGIS ST_DWithin 반경 검색
    (결과 < 3개면 반경 2× 자동 확장)
  → 카카오 PK6 민영 주차장 병합
  → Redis 실시간 잔여석 조회
  → 점수 랭킹
  → Claude Haiku 리뷰 분석 (timeout 15s + fallback)
  → Redis 캐싱
```

## AI 활용

**Claude Haiku 리뷰 분석** — 네이버·카카오 리뷰에서 주차 관련 감성, 시설 특징, 추천 여부를 추출합니다. Haiku를 선택한 이유는 Sonnet 대비 비용이 5~10배 저렴하면서 구조화된 JSON 출력에 충분한 성능을 보이기 때문입니다.

**pgvector RAG** — OpenAI `text-embedding-3-small`(1536-dim)으로 리뷰를 임베딩해 Supabase pgvector에 저장합니다. `/ask` 엔드포인트에서 의미 기반 검색 후 Claude Haiku가 인용 카드를 포함한 답변을 생성합니다. retrieval sources를 SSE로 먼저 전송해 인용 카드를 선렌더합니다.

**SSE 스트리밍** — `messages.stream` → `data: {chunk}` / 프론트 `ReadableStream.getReader()` + `AbortController`. Vercel 프록시 버퍼링 차단(`X-Accel-Buffering: no`)을 적용했습니다.

## 핵심 기술 결정

**3-레이어 폴백 Geocoding** — 단일 API에 의존하지 않고 네이버 → 카카오 → 지역검색 mapx/mapy로 폴백 체인을 구성합니다. NCloud Geocoding 210 endpoint 폐지를 추적하다 실제 운영 중 발견한 교훈입니다.

**Redis graceful degradation** — `_reset_redis()` 자동 재연결로 Redis가 다운돼도 캐시 없이 계속 응답합니다.

**pgbouncer + asyncpg 충돌 해결** — 실배포에서 "prepared statement already exists" 오류 발생. `NullPool` + `statement_cache_size=0` 조합으로 해결했습니다.

## 트러블슈팅

- NCloud Geocoding endpoint 폐지 → 추적 후 카카오 폴백 전환
- 공공데이터포털 HTTPS 차단 → HTTP 프록시 레이어 추가
- asyncpg Decimal/time 직렬화 오류 → 커스텀 JSON encoder
- 트렌드 비장소 오추출 → EXCLUDE_WORDS 130+ 단어 관리
