## 프로젝트 소개

핸드메이드 소품샵을 위한 풀스택 이커머스입니다.

상품 상세페이지를 만드는 작업이 가장 노동집약적이라는 문제의식에서 시작했습니다. 개발자 없이도 운영자가 직접 블록을 드래그해 상세페이지를 구성하고, 주문·CS까지 한 화면에서 처리할 수 있는 관리자 시스템을 목표로 했습니다.

## 주요 기능

**블록형 CMS 상세페이지 편집기** — `@dnd-kit` 기반 드래그 앤 드롭으로 텍스트·이미지·영상·구분선 블록을 자유롭게 조합합니다. Zod discriminated union + JSONB로 블록 타입을 타입 안전하게 저장합니다.

**Supabase RLS 2-role 인증** — 고객(customer)과 관리자(admin) 역할을 Row Level Security 정책으로 DB 레벨에서 분리합니다. SQL 정책 하나로 API 우회 시도를 차단합니다.

**PortOne V2 서버사이드 결제 검증** — 클라이언트 결제 완료 후 서버에서 PortOne API를 직접 호출해 금액·상태를 재검증합니다. 클라이언트 조작으로 결제 금액을 바꿔치기하는 공격을 서버에서 차단합니다.

**SECURITY DEFINER RPC 통계** — Supabase 함수에 `SECURITY DEFINER`를 적용해 집계 쿼리를 DB 내부에서 실행합니다. 클라이언트에 집계 로직을 노출하지 않으면서 권한 에스컬레이션도 방지합니다.

## 기술 결정

**XSS 방어** — 운영자 입력 HTML을 `DOMPurify` + URL allowlist로 정제합니다. 상세페이지는 운영자가 직접 HTML을 넣는 구조여서 서버 저장 전 한 번, 렌더링 전 한 번 총 2단계로 검증합니다.

**localStorage draft 자동저장** — 편집 중 새로고침이나 실수로 닫아도 초안이 남아있습니다. 저장 없이 이탈할 때 경고를 띄워 데이터 손실을 방지합니다.

**KST 수동 파싱** — Supabase의 타임존 기본값이 UTC이기 때문에 주문 생성일·통계 집계를 KST 기준으로 표시하려면 클라이언트에서 오프셋(`+09:00`)을 수동으로 처리합니다.
