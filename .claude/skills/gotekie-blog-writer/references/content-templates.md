## 1. 공통 마크다운 포맷

모든 글은 다음 메타데이터 블록으로 시작한다 (노션 API 호환):

```yaml
---
title: "[글 제목]"
description: "[한 줄 요약]"
tags: ["태그1", "태그2", "태그3"]
category: "리뷰 | 삽질기 | 튜토리얼 | 비교 | 회고"
date: "YYYY-MM-DD"
author: "고텍이 (Go Kyeongseok)"
thumbnail: "[썸네일 이미지 경로 또는 URL]"
series: "[시리즈명 (있을 경우)]"
---
```

### 공통 구조 규칙
- H1(#)은 글 제목으로 1번만 사용
- H2(##)로 대섹션 구분
- H3(###)로 소섹션 구분
- H4 이하는 가급적 사용하지 않음 (깊이 3단계까지)
- 코드 블록마다 언어 태그 필수
- 이미지 alt 텍스트 필수 작성

## 2. 신기술 리뷰 템플릿

```markdown
# [기술명] 써봤습니다 — [한 줄 감상]

> 📌 **한 줄 요약:** [이 글의 핵심 메시지]

안녕하세요, 고텍이예요! 오늘도 쉽게 가보자고!
[왜 이 기술에 관심을 가지게 됐는지 개인적 동기 1-2문장].
결론부터 말하면요, [핵심 결론 한 줄].
궁금하시면 같이 고!!

## 이게 뭔데요?

[기술 설명 3줄 이내. 비유를 활용해서 누구나 이해할 수 있게]

## 직접 써봤어요

### 환경 세팅

[설치/세팅 과정. 코드 블록 포함]

세팅은 이게 끝이에요. 베리 이지이지~

### 핵심 기능 체험

[실제로 써본 과정. 코드 + 설명 + 스크린샷]

### 성능은 어떨까?

[수치가 있으면 수치로. 체감이면 구체적 체감으로]

## 솔직한 평가

### 👍 좋았던 점
- [구체적 장점 1]
- [구체적 장점 2]

### 🤔 아쉬웠던 점
- [구체적 아쉬운 점 1]
- [구체적 아쉬운 점 2]

## 결론: 실무에서 쓸 만한가?

[명확한 판단 + 어떤 상황에서 추천하는지]

---

오늘도 읽어주셔서 감사해요!
여러분은 [이 기술] 써보셨나요? 경험 공유해주시면 저도 배울 수 있어서 좋겠어요.
다음에는 [다음 주제 예고]로 찾아올게요. 오늘도 이지하게, 고!! 🚀

기술을 만들고, 더 쉽게 삽시다! 🚀

#고텍이 #[기술태그] #[관련태그]
```

## 7. 노션 API 호환 메타데이터

노션 데이터베이스 프로퍼티 매핑:

| 블로그 필드 | 노션 프로퍼티 타입 | 비고 |
|---|---|---|
| title | Title | 필수 |
| description | Rich text | 검색/SEO용 |
| tags | Multi-select | 최대 5개 권장 |
| category | Select | 리뷰/삽질기/튜토리얼/비교/회고 |
| date | Date | ISO 8601 |
| author | Rich text | "고텍이 (Go Kyeongseok)" 고정 |
| status | Select | Draft / Published / Archived |
| series | Relation | 시리즈 연결 (선택) |
| thumbnail | Files & media | 썸네일 이미지 |

본문은 노션 블록 형태로 변환 가능:
- H2 → heading_2 블록
- H3 → heading_3 블록
- 코드 블록 → code 블록 (언어 태그 포함)
- 인용 → quote 블록
- 체크리스트 → to_do 블록
- 이미지 → image 블록 (URL 필요)
