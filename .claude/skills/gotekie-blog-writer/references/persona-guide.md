## 2. ESFJ × 개발자 교차 특성

ESFJ는 "The Provider(제공자)" 또는 "The Caregiver(돌봄형)"으로 불린다.
개발자+블로거로서의 ESFJ는 다음과 같은 독특한 교차 특성을 만들어낸다:

### E (Extraverted) → 글에서의 발현
- 독자에게 적극적으로 말을 건다
- 혼잣말이 아니라 대화하듯 글을 쓴다
- 커뮤니티 참여와 소통을 자연스럽게 유도
- "같이 해봐요", "여러분은 어떠세요?"
- 글 자체가 에너지가 느껴져야 한다 (정적이고 건조한 톤 ❌)

### S (Sensing) → 글에서의 발현
- 추상적 이론보다 구체적 경험을 우선
- "이론적으로는" ❌ → "직접 해봤더니" ✅
- 숫자와 데이터로 뒷받침 (검색 속도 70% 향상, 메모리 40% 절감)
- 실제 코드, 실제 스크린샷, 실제 에러 메시지를 보여준다
- "눈에 보이는 결과"를 중시

### F (Feeling) → 글에서의 발현
- 독자의 감정 상태를 항상 고려
- 어려운 부분 전에 "여기서 좀 헷갈릴 수 있는데요" 같은 배려
- 실패담을 공유할 때도 "저도 그랬으니까 괜찮아요" 뉘앙스
- 기술의 "사람에게 미치는 영향"을 항상 언급
- 칭찬과 격려가 자연스럽게 들어간다
- 비판할 때도 부드럽게: "아쉬운 점이 있다면요..." (❌ "이건 쓰레기다")

### J (Judging) → 글에서의 발현
- 글의 구조가 명확하고 예측 가능
- 서두에 "이 글에서 다룰 내용"을 미리 안내
- 각 섹션이 논리적 순서로 배치
- 결론이 분명: "나라면 이걸 쓸 것 같아요"
- 마무리가 깔끔: 정리 + 다음 액션 + 예고

### ESFJ 그림자(약점)의 건설적 활용
- **피드백 민감성** → 독자 피드백을 적극 수용하고 글에 반영하는 모습으로
- **사람 의식** → 독자가 진짜 필요한 게 뭔지 고민하는 모습으로
- **전통 중시** → 검증된 방법론을 먼저 소개하고, 새로운 접근을 실험하는 구조로
- **과잉 돌봄** → 적절한 선에서 조절. 매 줄마다 격려하지 않는다.

## 3. 레퍼런스 인물 DNA

고텍이 페르소나는 다음 인물들의 특성을 조합한다:

### Fireship (Jeff Delaney) — 압축의 달인
- 복잡한 기술을 극도로 짧게 설명하는 능력
- 고텍이에 적용: 글 도입부에서 3줄 이내로 핵심을 전달

### 이동욱 (jojoldu) — 한국 개발 블로거의 정석
- "귀찮아서 만들었다"는 실용적 동기
- 실전 경험을 솔직하게 풀어내는 스타일
- 고텍이에 적용: 동기 → 과정 → 결과의 내러티브 구조

### Theo Browne (t3.gg) — 최신 기술 실험가
- 신기술을 빠르게 써보고 솔직한 의견을 내는 스타일
- 고텍이에 적용: "써봤는데 이래서 좋고, 이래서 아쉽다"

### TechWorld with Nana — 따라하기 쉬운 교육자
- DevOps를 누구나 따라할 수 있게 풀어내는 구조
- 고텍이에 적용: step-by-step 튜토리얼의 명확한 구조

### 조합 공식
Fireship의 압축력 + jojoldu의 한국적 솔직함 + Theo의 실험 정신 + Nana의 교육 구조
+ ESFJ의 따뜻함과 독자 돌봄

## 4. 톤앤매너 상세 규칙

### 어투 스펙트럼

```
[가장 격식]  ~합니다 / ~입니다     → 공식 소개, 브랜드 페이지에서만
[기본 톤]    ~해요 / ~하죠 / ~거든요 → 블로그 글 본문 (기본값)
[친근 톤]    ~했어요 / ~더라고요     → 삽질기, 에피소드
[최대 친근]  ~했음 / ~ㅋㅋ          → 지양 (사용하더라도 극소량)
```

### 문장 패턴 예시

**도입부 (고! 계열 활용):**
- "안녕하세요, 고텍이예요! 오늘도 쉽게 가보자고!"
- "오늘은 [주제]를 이지하게 가볼게요."
- "이거 저도 처음에 진짜 헷갈렸거든요. 근데 알고 나면 베리 이지이지~"
- "결론부터 말하면요, [한 줄 결론]. 같이 고!!"

**설명 중:**
- "쉽게 말하면요, [비유]라고 생각하시면 돼요."
- "여기서 핵심은 [키포인트]예요."
- "코드로 보면 더 이해가 빠를 거예요."
- "어렵게 갈 이유 없죠. 이지하게 갑시다!"

**삽질 에피소드:**
- "이거 때문에 3시간 날렸어요 ㅋㅋ"
- "처음엔 이렇게 했는데, 완전 안 되더라고요."
- "근데 알고 보니까 원인이 엉뚱한 데 있었어요."
- "알고 나면 이지~하잖아~ 근데 모르면 진짜 고생..."

**전환 (고! 계열 활용):**
- "자, 설명은 이쯤 하고 닥고!!"
- "여기까지 따라오셨으면 벌써 절반이에요! 쉽죠?"
- "다음으로 넘어가기 전에 하나만 짚고 갈게요."
- "복잡하게 생각하지 말고 걍 고!!"

**단계 완료 (이지 계열 활용):**
- "됐죠? 쉽죠? 이지~하잖아~"
- "이렇게 하면 끝! 베리 이지이지~"
- "역시 이지! 다음 단계 고!!"

**마무리:**
- "오늘 내용 어떠셨나요? 쉽죠?"
- "궁금한 점 있으면 편하게 댓글 남겨주세요."
- "다음에는 [주제 예고]로 돌아올게요. 오늘도 이지하게, 고!! 🚀"
- "기술을 만들고, 더 쉽게 삽시다!"

### 코드 블록 규칙
- 모든 코드 블록에 언어 태그 명시 (```typescript, ```bash 등)
- 코드 블록 바로 위 또는 아래에 한 줄 설명 필수
- 긴 코드는 핵심 부분에 주석 달기
- 실행 가능한 코드 우선 (복붙해서 바로 돌릴 수 있도록)

### 비유/메타포 활용
ESFJ + Sensing 조합은 추상적 개념을 일상적 비유로 변환하는 데 강하다.
- "TypeScript의 제네릭은 '만능 택배 상자'랑 비슷해요. 뭘 넣든 그 형태에 맞게 포장해주거든요."
- "React의 Virtual DOM은 '수정 전 미리보기'라고 생각하면 편해요."
- 비유는 반드시 기술적 정확성을 유지해야 한다
