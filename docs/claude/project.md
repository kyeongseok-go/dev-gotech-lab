# 프로젝트 개요

- 프로젝트 목적: 개인 홈페이지 + 기술 블로그 + 서비스 플랫폼의 MVP를 1주 안에 배포한다.
- 핵심 사용자: 채용 담당자, 기술 리더, 개발자 커뮤니티 독자, 일반 방문자.
- 코드와 이력 관리는 GitHub를 기준으로 진행한다.
- 1차 목표: 홈 / 블로그 / 프로젝트 / 소개 / 구독 / 서비스 목록까지 배포 가능한 상태로 완성한다.
- 2차 목표: Supabase를 추가해 인증, 댓글, 구독 기능을 확장한다.
- 3차 목표: AI로 개발한 웹/앱 기능을 지속적으로 추가하는 AI Showcase 섹션을 운영한다.

## 기술 스택 고정

- 프론트엔드: Next.js 15 App Router + TypeScript + Tailwind CSS v4
- UI: shadcn/ui 기본, 필요한 경우 Aceternity UI를 제한적으로 사용
- 콘텐츠: MDX + Velite
- 배포: Cloudflare Pages + Workers + OpenNext adapter
- 데이터: D1 = 공개 서비스/캐시, R2 = 파일, Supabase = 인증/댓글/구독
- 버전 관리: GitHub
- 패키지 매니저: pnpm

## 출력 언어

- 모든 설명, 요약, TODO, 커밋 메시지 초안, 코드 주석은 기본적으로 한국어로 작성한다.
- 사용자에게 보여지는 UI 텍스트는 한국어를 기본으로 한다.
- 영어는 코드 식별자, 라이브러리 이름, 공식 고유명사에만 사용한다.

## 품질 목표

- 항상 동작하는 MVP를 우선한다.
- 과도한 추상화, 과도한 폴더 분리, 과도한 디자인 패턴 도입을 피한다.
- 초기 1주차에는 페이지 전환 애니메이션/3D보다 안정적인 배포와 콘텐츠 파이프라인을 우선한다.
- GitHub 저장소는 포트폴리오 관점에서 README, 태그, 릴리즈 노트, 이슈, PR 이력까지 관리한다.
